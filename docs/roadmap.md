# OpenDeck Browser Roadmap

## Phase 1 — Foundation

- Define project vision.
- Define MVP scope.
- Create repository documentation.
- Create AGENTS.md.
- Initialize the Tauri, React, TypeScript, and Rust app.
- Build the main layout.
- Build the basic tab system.
- Add internal views.

## Phase 2 — GitHub workflow

- Add secure GitHub connection.
- List repositories.
- Display repository metadata.
- Add issues view.
- Add pull requests view.
- Add external links to GitHub.
- Add loading, empty, and error states.

## Phase 3 — AI-assisted maintainer workflows

- Add manual AI summaries for issues.
- Add manual AI summaries for pull requests.
- Add privacy warning before sending content to an AI provider.
- Add local prompt templates.
- Add changelog draft generation.

## Phase 4 — Security hardening

- Review token storage.
- Review Tauri IPC boundaries.
- Validate external URLs.
- Review dependency risks.
- Add security documentation.
- Use Codex Security when available.

## Phase 5 — Open-source readiness

- Add issue templates.
- Add pull request template.
- Add contribution guide.
- Add code of conduct.
- Add security policy.
- Publish v0.1.0-alpha.
- Add screenshots or demo video.
- Start collecting feedback from contributors.

## Future ideas

- GitLab integration.
- Local project memory.
- Maintainer dashboard.
- Contributor onboarding mode.
- Dependency risk view.
- Release assistant.
- Documentation freshness checker.
- Plugin system.
