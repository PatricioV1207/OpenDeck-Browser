# OpenDeck Browser MVP

## MVP goal

The first MVP of OpenDeck Browser should prove that a maintainer-focused browser/workspace can improve the daily workflow of open-source maintainers.

The MVP must be small, secure, and easy to review.

## MVP features

1. Desktop application shell.
2. Main layout with left sidebar, top bar, central workspace, and optional bottom panel.
3. Basic tab system.
4. Internal views:
   - Home
   - Projects
   - Settings
   - About
5. Local project/workspace model.
6. Basic local settings.
7. GitHub connection flow.
8. Repository list.
9. Issues view.
10. Pull requests view.
11. Manual AI summary for issues and pull requests.
12. Initial security model.
13. Documentation for contributors.

## Out of scope for the MVP

- No custom browser engine.
- No extension marketplace.
- No cloud sync.
- No automatic execution of remote code.
- No automatic AI actions without user confirmation.
- No GitLab integration yet.
- No Jira integration yet.
- No mobile app.
- No organization/team management.
- No automatic issue or pull request editing.
- No background agent that changes repositories automatically.

## MVP success criteria

The MVP is successful when a user can:

1. Open OpenDeck Browser as a desktop app.
2. See a clean maintainer-focused interface.
3. Add or select a project workspace.
4. Connect to GitHub safely.
5. View repositories.
6. Open issues and pull requests.
7. Generate a manual AI summary for an issue or pull request.
8. Understand how to contribute to the project.
9. Review the security model from the documentation.
